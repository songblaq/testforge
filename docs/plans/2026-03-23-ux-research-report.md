# TestForge GUI UX 리서치 리포트

**작성일**: 2026-03-23
**분석 대상**: TestForge Web GUI (index.html, app.js, style.css)
**현재 평가**: 10/100 (Product Owner)
**분석 방법론**: 휴리스틱 평가 + 사용자 여정 매핑 + 정보 아키텍처 분석

---

## 1. 사용자 여정 맵 (User Journey Map)

### 1.1 페르소나 A: QA 엔지니어 (일일 사용자)

**프로필**: 테스트 설계/실행/리포트를 매일 수행하는 핵심 사용자. 효율성과 추적 가능성이 최우선.

| 단계 | 행동 | 보아야 할 것 (SEE) | 해야 할 것 (DO) | 알아야 할 것 (KNOW) | 현재 상태 |
|------|------|---------------------|------------------|----------------------|-----------|
| **1. 진입** | 프로젝트 선택 | 프로젝트별 진행 상태 요약, 마지막 작업 시간 | 프로젝트 선택/생성 | 어떤 프로젝트가 작업이 필요한지 | 카드에 badge만 있고 진행률/마지막 활동 없음 |
| **2. 현황 파악** | 대시보드 확인 | 전체 파이프라인 진행 상태, 미완료 항목 수, 실패한 테스트 | 한 눈에 전체 상태 확인 | 다음에 무엇을 해야 하는지 | context-bar에 숫자만 있고 대시보드 없음 |
| **3. 문서 업로드** | 요구사항 문서 업로드 | 업로드된 문서 내용 미리보기, 문서 간 관계 | 파일 업로드/삭제, 문서 내용 확인 | 어떤 문서가 분석에 사용되는지 | 파일 목록만 있고 내용 미리보기 없음 |
| **4. 분석 실행** | LLM 분석 실행 | 분석 진행 상태, 추출된 Feature/Persona/Rule 상세 | 분석 실행, 결과 편집/보완 | 분석 품질이 충분한지, 누락된 것은 없는지 | 테이블만 있고 편집 불가, 분석 진행 상태 없음 |
| **5. 테스트 케이스 생성** | 케이스 생성/편집 | 케이스-Feature 매핑, 커버리지 갭, 케이스 상세 | 케이스 생성, 수동 추가/편집/삭제, 우선순위 변경 | 어떤 Feature가 테스트되지 않았는지 | 생성만 가능, 수동 추가/편집/삭제 불가 |
| **6. 스크립트 생성** | Playwright 스크립트 생성 | 스크립트 코드 내용, 케이스-스크립트 매핑 | 스크립트 생성, 코드 확인/편집 | 스크립트가 올바른지, 어떤 케이스와 연결되는지 | 스크립트 수만 표시, 코드 보기/편집 불가 |
| **7. 테스트 실행** | 자동 테스트 실행 | 실시간 실행 로그, 결과 상세, 실행 이력 | 전체/선택 실행, 재실행, 실패 필터링 | 왜 실패했는지, 이전 실행과 비교 | 실행 이력 없음, 실시간 로그 없음, 선택 실행 불가 |
| **8. Manual QA** | 수동 체크리스트 수행 | 체크리스트 항목 상세, 이전 세션 결과, 진행률 | 패스/실패 마킹, 노트 작성, 스크린샷 첨부 | 어떤 항목이 남았는지, 이전에 어떤 결과였는지 | 이전 세션 이력 없음, 스크린샷 첨부 불가 |
| **9. 리포트** | 리포트 확인/공유 | 종합 리포트, 트렌드, 커버리지, 비교 데이터 | 리포트 생성, 다운로드, 공유 | 전체 품질 상태, 릴리스 준비도 | 리포트 로드만 가능, 다운로드/공유 없음, 트렌드 없음 |
| **10. 퇴장** | 작업 저장 및 종료 | 저장 확인, 다음 할 일 요약 | - | 작업이 저장되었는지 | 자동 저장 여부 불명확 |

**핵심 장벽**: 편집 불가, 실행 이력 없음, 엔티티 간 연결 불가시, 대시보드 부재

### 1.2 페르소나 B: 개발자 (비정기 사용자)

**프로필**: 테스트 실패 원인 파악, 커버리지 확인, 특정 Feature의 테스트 상태 확인이 목적.

| 단계 | 행동 | 보아야 할 것 (SEE) | 해야 할 것 (DO) | 알아야 할 것 (KNOW) | 현재 상태 |
|------|------|---------------------|------------------|----------------------|-----------|
| **1. 진입** | 특정 프로젝트 접근 | 최근 테스트 실패 요약, 자신의 Feature 상태 | 빠르게 프로젝트 선택 | 자신의 변경사항이 테스트에 영향을 줬는지 | 프로젝트 카드에 실패 정보 없음 |
| **2. 실패 분석** | 실패한 테스트 확인 | 실패 원인 (에러 메시지, 스택 트레이스, 스크린샷) | 실패 케이스 필터링, 상세 로그 확인 | 어떤 코드 변경이 실패를 일으켰는지 | Execution 탭에 출력 200자 잘림, 스택 트레이스 없음 |
| **3. 커버리지 확인** | Feature별 커버리지 | Feature-케이스 매핑 시각화, 커버리지 퍼센트 | Feature별 필터링, 갭 확인 | 내 Feature가 충분히 테스트되고 있는지 | Report 탭에서만 숫자로 확인, 시각화 없음 |
| **4. 스크립트 확인** | 테스트 코드 검토 | 스크립트 소스 코드, 케이스와의 매핑 | 코드 읽기, 수정 제안 | 테스트가 올바르게 작성됐는지 | 스크립트 내용 볼 수 없음 |
| **5. 퇴장** | 필요한 정보 획득 | 요약 정보 | 관련 정보 공유/내보내기 | 문제 해결에 필요한 정보를 얻었는지 | 내보내기 기능 없음 |

**핵심 장벽**: 실패 원인 상세 부족, 스크립트 코드 열람 불가, Feature-테스트 매핑 시각화 없음

---

### 1.3 페르소나 C: 프로젝트 매니저 (리포트 열람자)

**프로필**: 릴리스 품질 판단, 진행 상태 추적, 팀에 보고하는 것이 목적.

| 단계 | 행동 | 보아야 할 것 (SEE) | 해야 할 것 (DO) | 알아야 할 것 (KNOW) | 현재 상태 |
|------|------|---------------------|------------------|----------------------|-----------|
| **1. 진입** | 프로젝트 현황 확인 | 전체 프로젝트 대시보드 (녹/적/황 신호등) | 프로젝트 선택 | 어떤 프로젝트에 주의가 필요한지 | 프로젝트 카드에 건강 상태 표시 없음 |
| **2. 진행 상태** | 파이프라인 진행률 | 각 단계(입력->분석->케이스->스크립트->실행) 진행률 | 전체 현황 한 눈에 파악 | 팀이 어디까지 진행했는지 | context-bar에 숫자만 있음, 파이프라인 시각화 없음 |
| **3. 품질 지표** | 테스트 결과 요약 | 통과율, 실패 트렌드, 커버리지 추이 | 대시보드 확인, 기간별 필터링 | 품질이 개선되고 있는지, 릴리스 가능한지 | 트렌드 없음, 현재 실행 결과만 있음 |
| **4. 리포트 공유** | 리포트 생성/내보내기 | PDF/HTML 리포트, 차트, 요약 | 리포트 다운로드, 이메일/슬랙 공유 | 리포트가 적절한 정보를 담고 있는지 | Markdown 렌더링만 있고, 다운로드/공유 없음 |
| **5. 퇴장** | 의사결정 완료 | 의사결정에 필요한 요약 | Go/No-Go 판단 | 릴리스 준비가 됐는지 | 판단 근거 시각화 없음 |

**핵심 장벽**: 대시보드 없음, 트렌드 데이터 없음, 리포트 다운로드/공유 불가, 품질 지표 시각화 없음

## 2. 탭별 Pain Point 분석

### 2.1 Inputs 탭

| 항목 | 내용 |
|------|------|
| **사용자 기대** | 문서를 업로드하고, 내용을 미리보고, 어떤 문서가 어떤 분석에 사용되는지 확인할 수 있다. 문서를 편집/교체할 수 있다. |
| **실제 제공** | 파일 이름/타입/크기의 리스트. 업로드와 삭제만 가능. 상세 패널에서도 파일명/타입/크기만 반복 표시. |
| **갭** | (1) 문서 내용 미리보기 없음 -- **심각도: HIGH** (2) 문서 간 관계/의존성 표시 없음 -- **심각도: MEDIUM** (3) 파일 교체 (같은 이름 덮어쓰기) 기능 없음 -- **심각도: LOW** (4) 드래그로 순서 변경 불가 -- **심각도: LOW** |

### 2.2 Analysis 탭

| 항목 | 내용 |
|------|------|
| **사용자 기대** | 분석 결과를 확인하고, 잘못 추출된 항목을 수정/삭제하고, 누락된 Feature를 수동 추가할 수 있다. 분석 진행 상태를 실시간으로 볼 수 있다. |
| **실제 제공** | Feature/Persona/Rule의 읽기 전용 테이블. 행 클릭 시 상세 패널에서 추가 필드 확인 가능. 편집/추가/삭제 불가. |
| **갭** | (1) Feature/Persona/Rule 편집 불가 -- **심각도: CRITICAL** (2) 수동 추가 불가 -- **심각도: CRITICAL** (3) 삭제 불가 -- **심각도: HIGH** (4) 분석 실행 중 진행 상태(프로그레스 바, 단계 표시) 없음 -- **심각도: HIGH** (5) Feature-문서 소스 역추적 불가 -- **심각도: MEDIUM** (6) 분석 히스토리 (이전 분석과 비교) 없음 -- **심각도: MEDIUM** |

### 2.3 Cases 탭

| 항목 | 내용 |
|------|------|
| **사용자 기대** | 테스트 케이스를 확인하고, 수동으로 추가/편집/삭제할 수 있다. Feature별/우선순위별 필터링이 가능하다. 케이스-Feature 커버리지 매트릭스를 볼 수 있다. |
| **실제 제공** | 생성된 케이스의 읽기 전용 테이블. 텍스트 필터와 타입 필터 있음. 행 클릭 시 상세 패널에서 steps/preconditions 등 확인 가능. |
| **갭** | (1) 케이스 수동 추가 불가 -- **심각도: CRITICAL** (2) 케이스 편집 불가 -- **심각도: CRITICAL** (3) 케이스 삭제 불가 -- **심각도: CRITICAL** (4) 우선순위별 필터 없음 (타입 필터만 있음) -- **심각도: HIGH** (5) Feature별 필터/그룹핑 없음 -- **심각도: HIGH** (6) 케이스-Feature 커버리지 매트릭스 없음 -- **심각도: HIGH** (7) 케이스 상태 변경 불가 (pending -> approved 등) -- **심각도: MEDIUM** (8) 케이스 내보내기(CSV/JSON) 없음 -- **심각도: MEDIUM** (9) 벌크 작업 (다중 선택 삭제/상태 변경) 없음 -- **심각도: MEDIUM** |

### 2.4 Scripts 탭

| 항목 | 내용 |
|------|------|
| **사용자 기대** | 생성된 Playwright 스크립트의 코드를 확인하고, 어떤 케이스와 연결되는지 보고, 개별 스크립트를 다운로드하거나 편집할 수 있다. |
| **실제 제공** | 스크립트 개수만 표시 ("N scripts"). 상세 패널에서도 파일명과 크기만 표시. 코드 내용 없음. |
| **갭** | (1) 스크립트 코드 내용 표시 없음 -- **심각도: CRITICAL** (2) 스크립트-케이스 매핑 표시 없음 -- **심각도: CRITICAL** (3) 개별 스크립트 다운로드 불가 -- **심각도: HIGH** (4) 스크립트 전체 다운로드(ZIP) 불가 -- **심각도: HIGH** (5) 스크립트 편집 불가 -- **심각도: MEDIUM** (6) 스크립트 재생성 (특정 케이스만) 불가 -- **심각도: MEDIUM** |

### 2.5 Execution 탭

| 항목 | 내용 |
|------|------|
| **사용자 기대** | 테스트를 실행하고, 실시간 진행 상태를 보고, 실패 원인을 분석하고, 이전 실행과 비교할 수 있다. 개별 테스트를 재실행할 수 있다. |
| **실제 제공** | Total/Passed/Failed 숫자와 프로그레스 바. 결과 테이블에 case_id, status, duration, output(200자 잘림). |
| **갭** | (1) 실행 이력 없음 (현재 결과만 표시) -- **심각도: CRITICAL** (2) 실시간 실행 로그/진행 없음 (완료까지 스피너만) -- **심각도: CRITICAL** (3) 실패 상세 (스택 트레이스, 스크린샷) 없음 -- **심각도: CRITICAL** (4) 개별/선택 테스트 실행 불가 -- **심각도: HIGH** (5) 이전 실행과 결과 비교 불가 -- **심각도: HIGH** (6) 실패한 테스트만 재실행 불가 -- **심각도: HIGH** (7) 실행 결과 필터링 (실패만/통과만) 없음 -- **심각도: MEDIUM** (8) 테스트 결과 내보내기 없음 -- **심각도: MEDIUM** |

### 2.6 Manual QA 탭

| 항목 | 내용 |
|------|------|
| **사용자 기대** | 수동 테스트 체크리스트를 수행하고, 스크린샷/증거를 첨부하고, 이전 세션과 비교하고, 세션 이력을 관리할 수 있다. 자동 테스트 결과와 통합된 뷰를 볼 수 있다. |
| **실제 제공** | 세션 시작/종료. 항목별 Pass/Fail 마킹과 텍스트 노트. 프로그레스 바와 카운터. |
| **갭** | (1) 이전 세션 이력/목록 없음 -- **심각도: CRITICAL** (2) 스크린샷/파일 첨부 불가 -- **심각도: HIGH** (3) 자동 테스트 결과와의 통합 뷰 없음 -- **심각도: HIGH** (4) 체크리스트 항목 수동 추가/편집 불가 -- **심각도: HIGH** (5) 세션 이름/설명 설정 불가 -- **심각도: MEDIUM** (6) 완료된 세션 재열기/수정 불가 -- **심각도: MEDIUM** (7) 체크리스트 항목 순서 변경 불가 -- **심각도: LOW** |

### 2.7 Report 탭

| 항목 | 내용 |
|------|------|
| **사용자 기대** | 종합적인 테스트 리포트를 보고, 차트와 그래프로 시각화하고, PDF/HTML로 다운로드하고, 팀에 공유할 수 있다. 시간별 트렌드를 확인할 수 있다. |
| **실제 제공** | Markdown 또는 HTML 형식 선택 후 "Load Report" 클릭. Markdown은 간단한 파서로 렌더링. Feature/Rule 커버리지 퍼센트 2개 숫자. |
| **갭** | (1) 리포트 내용이 실질적으로 비어 있거나 부실함 -- **심각도: CRITICAL** (2) 차트/그래프 시각화 없음 -- **심각도: CRITICAL** (3) 리포트 다운로드 (PDF/HTML 파일) 불가 -- **심각도: CRITICAL** (4) 시간별 트렌드/비교 없음 -- **심각도: HIGH** (5) 리포트 공유 기능 없음 -- **심각도: HIGH** (6) 리포트 커스터마이징 불가 -- **심각도: MEDIUM** (7) 자동 테스트 + Manual QA 통합 리포트 없음 -- **심각도: MEDIUM** |

### 3.4 Test Case (테스트 케이스)

| 인터랙션 | 현재 상태 | 요구 구현 |
|-----------|-----------|-----------|
| **목록 보기** | 테이블 + 텍스트 필터 + 타입 드롭다운 | 우선순위 필터, Feature 필터, 상태 필터 추가. 다중 정렬. 페이지네이션 |
| **상세 보기** | 행 클릭 -> 사이드 패널 (steps/preconditions/expected_result/tags/rule_ids 포함) | 관련 Feature 링크(클릭 가능), 관련 Script 링크, 실행 결과 이력 추가 |
| **생성/추가** | 불가 | "Add Case" 버튼 -> 폼 모달. 타이틀/설명/Feature 연결/우선순위/타입/steps 편집기 |
| **편집/수정** | 불가 | 상세 패널에서 모든 필드 인라인 편집. Steps 추가/삭제/순서변경 |
| **삭제** | 불가 | 개별 삭제 + 벌크 삭제 (체크박스 다중 선택) |
| **연결** | feature_id 텍스트만 표시 | Feature 링크 클릭 -> Feature 상세로 이동. Script 링크. Execution 결과 링크 |
| **내보내기** | 불가 | JSON/CSV/Excel 다운로드. 필터된 결과만 내보내기 옵션 |

### 3.5 Test Script (테스트 스크립트)

| 인터랙션 | 현재 상태 | 요구 구현 |
|-----------|-----------|-----------|
| **목록 보기** | 스크립트 수만 표시 (개별 파일 목록 없음) | 개별 스크립트 파일 목록 (파일명, 크기, 매핑된 케이스 수) |
| **상세 보기** | 파일명/크기만 표시 | 코드 뷰어 (구문 강조), 매핑된 테스트 케이스 목록, 마지막 실행 결과 |
| **생성/추가** | "Generate Scripts" 버튼으로 전체 생성 | 개별 케이스별 스크립트 재생성, 수동 스크립트 추가(파일 업로드) |
| **편집/수정** | 불가 | 코드 에디터 (기본적인 텍스트 편집) 또는 외부 에디터로 열기 링크 |
| **삭제** | 불가 | 개별 스크립트 삭제 |
| **연결** | 없음 | Script -> Test Case(s) 매핑 표시, Script -> Execution Result 링크 |
| **내보내기** | 불가 | 개별 파일 다운로드, 전체 ZIP 다운로드 |

### 3.6 Execution Result (실행 결과)

| 인터랙션 | 현재 상태 | 요구 구현 |
|-----------|-----------|-----------|
| **목록 보기** | 현재 실행의 결과만 테이블로 표시 | 실행 이력 목록 (날짜별), 각 실행의 요약 (통과/실패/건너뛰기 수) |
| **상세 보기** | 행 클릭 -> 사이드 패널 (output/stderr/return_code) | 전체 로그 뷰, 스크린샷 뷰, 스택 트레이스 하이라이팅, 관련 케이스/스크립트 링크 |
| **생성/추가** | "Run Tests" 버튼으로 전체 실행 | 선택 실행 (체크박스), 실패만 재실행, 태그별 실행 |
| **편집/수정** | 해당 없음 | 실행 결과에 노트/코멘트 추가 기능 |
| **삭제** | 불가 | 이전 실행 이력 삭제 |
| **연결** | case_id 텍스트만 표시 | Case 링크 (클릭 -> Case 상세), Script 링크 |
| **내보내기** | 불가 | 실행 결과 JSON/JUnit XML 다운로드 |

### 3.7 Manual QA Session (수동 QA 세션)

| 인터랙션 | 현재 상태 | 요구 구현 |
|-----------|-----------|-----------|
| **목록 보기** | 현재 세션만 표시 (세션 목록 없음) | 세션 이력 목록 (날짜, 진행률, 결과 요약) |

| **내보내기** | 불가 | 세션 결과 JSON/PDF 다운로드 |

## 4. 정보 아키텍처 권장안

### 4.1 현재 구조의 문제점

```
현재: 7개 탭이 독립적인 사일로(silo)로 존재
[Inputs] [Analysis] [Cases] [Scripts] [Execution] [Manual QA] [Report]
    |          |          |        |         |            |          |
  (단방향 흐름만 존재, 탭 간 연결 없음)
```

**핵심 문제들**:
1. **데이터 흐름 불가시**: 사용자가 "이 Feature에 대한 테스트 케이스는 무엇인가?"를 답할 수 없음
2. **선형적 사일로**: 각 탭이 독립적이라 엔티티 간 관계를 추적할 수 없음
3. **대시보드 부재**: 전체 현황을 한 눈에 파악할 수 없음
4. **히스토리 부재**: 시간축 데이터가 없어 변화 추적 불가

### 4.2 권장 정보 아키텍처

```
[Dashboard] -- 최상위 레벨, 전체 파이프라인 현황
    |
    +-- [Inputs] -- 문서 관리 (미리보기 포함)
    |       |
    |       v  (분석 소스)
    +-- [Analysis] -- Feature / Persona / Rule 관리
    |       |
    |       v  (케이스 소스)
    +-- [Test Design] -- Cases + Coverage Matrix
    |       |
    |       v  (스크립트 소스)
    +-- [Automation] -- Scripts + Execution (통합)
    |       |
    |       v  (실행 결과)
    +-- [Manual QA] -- 수동 테스트 (세션 이력 포함)
    |       |
    |       v  (수동 결과)
    +-- [Reports] -- 통합 리포트 + 트렌드
```

### 4.3 탭 간 데이터 흐름 설계

#### 4.3.1 상향 추적 (Upstream Traceability)
어떤 엔티티에서든 그 소스로 거슬러 올라갈 수 있어야 한다:
- **Test Case** -> 소스 Feature -> 소스 Document
- **Script** -> 소스 Test Case -> 소스 Feature
- **Execution Result** -> 실행한 Script -> 소스 Test Case -> 소스 Feature
- **Manual QA Item** -> 소스 Test Case (있는 경우)

#### 4.3.2 하향 추적 (Downstream Traceability)
어떤 엔티티에서든 그 결과물/영향으로 내려갈 수 있어야 한다:
- **Document** -> 추출된 Features
- **Feature** -> 생성된 Test Cases -> 생성된 Scripts -> Execution Results
- **Business Rule** -> 관련 Test Cases -> Execution Results

#### 4.3.3 각 탭에서 보여줘야 할 교차 정보

| 탭 | 현재 탭의 데이터 | 추가로 보여줘야 할 교차 데이터 |
|----|-------------------|-------------------------------|
| **Dashboard** (신규) | 전체 요약 | 파이프라인 단계별 진행률, 최근 실행 결과, 실패 알림, 커버리지 게이지 |
| **Inputs** | 문서 목록 | 각 문서에서 추출된 Feature 수 (분석 완료 시) |
| **Analysis** | Features/Personas/Rules | 각 Feature의 테스트 케이스 수, 커버리지 상태 (빨강/초록 표시) |
| **Test Design** | Test Cases | 각 케이스의 Feature 링크, 실행 결과 상태, Script 존재 여부 |
| **Automation** | Scripts + Execution | 각 스크립트의 매핑된 케이스, 최근 실행 결과. 실행 이력 타임라인 |
| **Manual QA** | 세션/체크리스트 | 각 항목의 관련 Test Case (자동 테스트 결과와 대조) |
| **Reports** | 리포트 내용 | 자동 + 수동 통합 결과, 커버리지 매트릭스, 시계열 트렌드 차트 |

### 4.5 상세 패널 개선 권장

현재 상세 패널은 읽기 전용 필드만 나열한다. 다음으로 개선해야 한다:

1. **편집 모드**: 각 필드 옆에 편집 아이콘. 클릭 시 인라인 편집 활성화
2. **관련 엔티티 섹션**: 패널 하단에 "Related Items" 영역
   - Feature 상세 -> "Test Cases for this Feature" 리스트 (클릭 가능)
   - Case 상세 -> "Source Feature" 링크 + "Script" 링크 + "Execution History" 리스트
3. **액션 버튼**: 패널 상단/하단에 컨텍스트에 맞는 액션
   - Feature 상세 -> "Generate Cases for this Feature" 버튼
   - Case 상세 -> "Generate Script" / "Run This Case" 버튼
   - Execution 상세 -> "Re-run" / "View Script" 버튼
4. **히스토리 탭**: 패널 내 탭으로 해당 엔티티의 변경/실행 이력

## 5. 우선순위별 개선 로드맵

### Phase 1: 기반 (사용 가능하게 만들기) -- CRITICAL

1. **CRUD 완성**: 모든 엔티티(Feature, Persona, Rule, Case)에 추가/편집/삭제 기능
2. **스크립트 코드 뷰어**: Scripts 탭에서 실제 코드 내용을 구문 강조와 함께 표시
3. **실행 이력**: Execution 탭에 이전 실행 목록과 결과 보존
4. **실패 상세**: 실행 결과에서 전체 출력/에러/스택 트레이스 표시
5. **리포트 다운로드**: 생성된 리포트를 Markdown/HTML 파일로 다운로드

### Phase 2: 연결 (데이터 흐름 가시화) -- HIGH

6. **엔티티 간 링크**: Feature <-> Case <-> Script <-> Result 양방향 네비게이션
7. **Dashboard 탭**: 파이프라인 진행 상태, 최근 활동, 핵심 지표 요약
8. **커버리지 매트릭스**: Feature x Test Case 매트릭스 시각화
9. **Manual QA 세션 이력**: 이전 세션 목록과 결과 보존
10. **문서 미리보기**: Inputs 탭에서 문서 내용 미리보기

### Phase 3: 효율 (워크플로 최적화) -- MEDIUM

11. **벌크 작업**: 다중 선택 삭제, 상태 일괄 변경
12. **고급 필터**: 우선순위/Feature/상태별 다중 필터, 정렬
13. **내보내기**: 모든 엔티티 JSON/CSV 내보내기
14. **LLM 작업 진행 상태**: 분석/생성 중 실시간 프로그레스 표시
15. **선택 실행**: 특정 케이스/스크립트만 선택하여 실행

### Phase 4: 인사이트 (의사결정 지원) -- LONG-TERM

## 7. 결론

TestForge GUI는 현재 **"읽기 전용 파이프라인 뷰어"** 수준이다. QA 자동화 플랫폼으로서 기본적으로 필요한 세 가지 핵심 기능이 모두 부재하다:

1. **조작 가능성 (Manipulability)**: 사용자가 데이터를 만들고, 고치고, 지울 수 없다
2. **추적 가능성 (Traceability)**: 엔티티 간의 관계를 따라갈 수 없다
3. **시간 가능성 (Temporality)**: 이전 결과와 비교하거나 변화를 추적할 수 없다

Phase 1의 CRITICAL 항목들을 먼저 구현하면 10/100 -> 40~50/100 수준으로 올릴 수 있고, Phase 2까지 완료하면 70/100 이상의 사용 가능한 도구가 될 수 있다.
