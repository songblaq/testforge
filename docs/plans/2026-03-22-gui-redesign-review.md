# TestForge GUI 재설계 리뷰 — 2026-03-22

## 참석
- Designer (Opus) — UI/UX, 정보 아키텍처
- Analyst (Opus) — 제품 기획, 페르소나별 시나리오
- Critic (Opus) — 기획 비판, 대안 아키텍처

## 핵심 문제 (전원 합의)

### CRITICAL
1. **프로젝트 컨텍스트 부재** — 탭 전환 시 어떤 프로젝트 데이터인지 표시 없음
2. **파일 업로드 UI 없음** — 파이프라인 시작점(문서 입력)이 GUI에서 불가능
3. **Manual QA 탭 누락** — 백엔드 4개 API 완비, 프론트엔드 0줄
4. **Scripts 탭 누락** — 백엔드 API 존재, 프론트엔드 없음
5. **케이스/분석 편집 불가** — PUT API 존재하나 프론트엔드 미연결

### HIGH
6. 기획 구조가 QA 워크플로우와 불일치 (기능 중심 → 프로젝트 중심 필요)
7. 파이프라인 진행도 표시 없음
8. 단계 의존성 가드 없음
9. 반응형 미대응
10. Markdown 보고서가 monospace 텍스트로 표시

## 합의된 새 화면 구조

```
+------------------------------------------------------+
| TestForge          [Project: my-webapp ▼]    [+ New]  |  ← 프로젝트 드롭다운

+------------------------------------------------------+
```
