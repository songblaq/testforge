# TestForge Configuration Reference

TestForge uses YAML configuration files at two levels:

- **Project config** — `.testforge/config.yaml` inside each project directory
- **Global config** — `~/.testforge/config.yaml` (optional; applies to all projects)

Project config always takes precedence over global config.

---

## Project Config: `.testforge/config.yaml`

Created automatically by `testforge init`. Edit it to customize a project.

### Full Schema

```yaml
# ── Identity ──────────────────────────────────────────────────────────────────
project_name: my-webapp        # Display name for the project
version: "0.1.0"               # Semantic version of your test suite

# ── LLM Provider ──────────────────────────────────────────────────────────────
llm_provider: anthropic        # Provider: anthropic | openai | cli | ollama
llm_model: ""                  # Model name; empty = use provider default
                               # anthropic default: claude-3-5-sonnet-20241022
                               # openai default:    gpt-4o
                               # cli:               uses the claude/codex binary
                               # ollama example:    qwen3.5:35b-a3b

# ── Directory Layout (relative to project root) ────────────────────────────────
input_dir: inputs              # Source documents placed here
output_dir: output             # Reports written here
evidence_dir: evidence         # Screenshots and logs from test runs
cases_dir: cases               # Generated test cases (cases.json)
analysis_dir: analysis         # Analysis output (analysis.json)

# ── Extra (arbitrary key/value, accessible to custom scripts) ─────────────────
extra: {}
```

### Field Reference

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `project_name` | string | (directory name) | Human-readable project name |
| `version` | string | `"0.1.0"` | Version of the test suite |
| `llm_provider` | string | `"anthropic"` | LLM backend to use |
| `llm_model` | string | `""` | Specific model; empty uses provider default |
| `input_dir` | string | `"inputs"` | Directory for source documents |
| `output_dir` | string | `"output"` | Directory for generated reports |
| `evidence_dir` | string | `"evidence"` | Directory for collected evidence |
| `cases_dir` | string | `"cases"` | Directory for test case JSON |
| `analysis_dir` | string | `"analysis"` | Directory for analysis JSON |
| `extra` | mapping | `{}` | Arbitrary key/value for custom use |

---

## Global Config: `~/.testforge/config.yaml`

Applies defaults across all projects. Uses the same schema as project config. Any key set here
is used as the fallback when the project config does not specify it.

```bash
mkdir -p ~/.testforge
cat > ~/.testforge/config.yaml <<'EOF'
llm_provider: anthropic
llm_model: claude-3-5-sonnet-20241022
EOF
```

## LLM Provider Setup

### Anthropic Claude

**Install:**
```bash
pip install "testforge[anthropic]"
```

**Configure:**
```bash
export ANTHROPIC_API_KEY="sk-ant-..."
```

**Config:**
```yaml
llm_provider: anthropic
llm_model: ""                        # default: claude-3-5-sonnet-20241022
# llm_model: claude-3-opus-20240229  # use Opus for deeper analysis
# llm_model: claude-3-haiku-20240307 # use Haiku for faster/cheaper runs
```

---

### OpenAI GPT

**Install:**
```bash
pip install "testforge[openai]"
```

**Configure:**
```bash
export OPENAI_API_KEY="sk-..."
```

**Config:**
```yaml
llm_provider: openai
llm_model: ""        # default: gpt-4o
# llm_model: gpt-4o-mini   # cheaper/faster
# llm_model: o1-preview     # reasoning model
```

---

### CLI Tools (claude / codex)

Use an installed CLI binary instead of the API. No API key needed if the CLI handles auth itself.

**Requirements:**
- The binary (`claude` or `codex`) must be in your `PATH`

**Config:**
```yaml
llm_provider: cli
llm_model: claude     # or: codex
```

---

### Ollama (Local LLM)

For fully local inference with no external API calls.

**Requirements:**
- Ollama running at `http://localhost:11434` (or set `OLLAMA_HOST`)
- The model pulled: `ollama pull qwen3.5:35b-a3b`

**Config:**
```yaml
llm_provider: ollama
llm_model: qwen3.5:35b-a3b
```

## Connector Configuration

Connectors are configured inside test scripts generated by `testforge script`, or can be
instantiated directly in custom scripts.

### Browser (Playwright)

```python
from testforge.execution.connectors.browser import BrowserConnector

async with BrowserConnector(headless=True, browser="chromium") as conn:
    result = await conn.run(script_path="scripts/login_test.py")
```

Configuration options:

| Option | Default | Description |
|--------|---------|-------------|
| `headless` | `True` | Run browser in headless mode |
| `browser` | `"chromium"` | Browser engine: `chromium`, `firefox`, `webkit` |
| `timeout` | `30000` | Page timeout in milliseconds |
| `base_url` | `""` | Base URL prepended to relative paths |

---

### HTTP (httpx)

```python
from testforge.execution.connectors.http import HTTPConnector

conn = HTTPConnector(base_url="https://api.example.com", timeout=10)
result = await conn.get("/health")
```

Configuration options:

| Option | Default | Description |
|--------|---------|-------------|
| `base_url` | `""` | Base URL for all requests |
| `timeout` | `30` | Request timeout in seconds |
| `headers` | `{}` | Default headers for every request |
| `verify_ssl` | `True` | Verify TLS certificates |

---

### Shell (subprocess)

```python
from testforge.execution.connectors.shell import ShellConnector

conn = ShellConnector(cwd="/path/to/project")
result = conn.run("npm test")
```

Configuration options:

| Option | Default | Description |
|--------|---------|-------------|
| `cwd` | current dir | Working directory for commands |
| `timeout` | `60` | Command timeout in seconds |
| `env` | `{}` | Extra environment variables |
| `shell` | `True` | Use shell expansion |

---

### SSH

```python
from testforge.execution.connectors.ssh import SSHConnector

conn = SSHConnector(host="192.168.1.10", user="deploy", key_path="~/.ssh/id_ed25519")
result = conn.run("systemctl status myapp")
```

Configuration options:

| Option | Default | Description |
|--------|---------|-------------|
| `host` | _(required)_ | Remote hostname or IP |
| `port` | `22` | SSH port |
| `user` | _(required)_ | SSH username |
| `key_path` | `"~/.ssh/id_rsa"` | Path to private key |
| `timeout` | `30` | Connection timeout in seconds |

---

## Environment Variables

| Variable | Description |
|----------|-------------|
| `ANTHROPIC_API_KEY` | Anthropic API key |
| `OPENAI_API_KEY` | OpenAI API key |
| `OLLAMA_HOST` | Ollama base URL (default: `http://localhost:11434`) |
| `TESTFORGE_CONFIG` | Override path to global config file |
