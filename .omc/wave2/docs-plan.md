## 2. README fixes (from diagnostics + audit)

### 2.1 “What is TestForge?” / opening claims

- **Current issue:** Paragraph claims “Playwright browser scripts, HTTP API tests, shell-command runners” as part of the shipped full pipeline; implementation is **Playwright-focused** for generated scripts and default `run` (pytest). HTTP/Shell are **not** generated script types today (`src/testforge/scripts/generator.py` registers **playwright only**).
- **Fix:** State that the default path is **Playwright (Python/pytest) scripts**; describe HTTP API / shell execution as **planned** or extension/connectors, consistent with the feature list and pipeline diagram.

### 2.2 Key Features list

- Align bullets with code: **script generation = Playwright only** (already partially correct at line 38); tighten line 40–41 so “Flexible execution” does not imply shipped HTTP/Shell runners—use “planned” or “connectors (partial / placeholder)” per `README` connectors table and `runner` behavior.
- **Ollama:** Change from implied “alongside others” to **explicitly supported** via `llm_provider: ollama` and `OllamaAdapter` (`src/testforge/llm/ollama.py`)—remove any “coming soon” tone (CHANGELOG Notes currently contradict this).

### 2.3 Pipeline ASCII diagram (lines 50–61)

- **Issue:** Scripts column shows “HTTP / Shell”; execution shows “API / Shell / SSH” as peers of Browser—misleading for default CLI (`testforge run` → pytest on `*.py` scripts).
- **Fix:** Redraw or relabel so **Scripts** emphasizes **Playwright (Python)**; execution column shows **pytest / Playwright** as default; add a footnote line under the diagram: *Default `testforge run` executes generated Playwright-based Python tests via pytest; HTTP/Shell/SSH connector tracks are planned or extension points.*

### 2.4 Installation (Quick Start)

- Add: `pip install "testforge[web]"` next to other extras (and mention it again in the new Web GUI section).

### 2.5 CLI Reference table (lines 137–152)

Add rows:

### 2.6 Connectors section (lines 239–248)

- **SSH:** Already marked placeholder—keep.
- **HTTP / Shell:** If connectors exist in code but are not the default `run` path, say so in one sentence to avoid the “we generate and run HTTP/shell tests” implication.

### 2.7 Project Structure (`scripts/` line)

- If only `.py` Playwright tests are generated by default, change `Generated automation scripts (.py, .sh)` to **`.py` (Playwright/pytest)** unless `.sh` is actually produced—verify before edit.

## 3. Exact README content — **Web GUI** section

**Placement:** After **TUI Mode** (after line ~206) or before **Configuration**—recommended **after TUI** to keep “interactive UIs” together.

**Paste as-is:**

```markdown
## Web GUI

Run TestForge in the browser with the FastAPI stack (optional extra):

```bash
pip install "testforge[web]"
testforge web
```

The server defaults to `http://127.0.0.1:8000`. Override host and port:

```bash
testforge web --host 0.0.0.0 --port 8080
```

If FastAPI or Uvicorn are missing, the CLI exits with a hint to run `pip install 'testforge[web]'`.
```
